# Project Instructions

## Documentation
- Write documentation in the author's voice.
- Use a concise, academic tone.
- Avoid boilerplate and unnecessary explanation.

## Product Direction
- Design the character customization system as a reusable Unity 6.3 package.
- Keep eventual Unity Asset Store distribution possible.
- Avoid assumptions that bind the package to a single game.

## Scope Control
- Implement only the requested behaviour.
- Do not introduce speculative features or abstractions.
- Ask for approval before expanding scope or adding architectural complexity.
